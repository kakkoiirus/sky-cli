# Product Requirements Document (PRD): `sky`

## 1. Обзор проекта (Overview)
**Название:** sky
**Тип продукта:** CLI утилита (Command Line Interface)
**Цель:** Предоставить пользователю быстрый и удобный способ узнать текущую погоду в терминале без необходимости регистрации и получения API-ключей.
**Краткое описание:** Легковесная кроссплатформенная утилита на языке Go, использующая Open-Meteo API.

---

## 2. Технический стек (Tech Stack)
*   **Язык разработки:** Go (Golang).
*   **API данных:** Open-Meteo (без ключа, HTTPS).
*   **Поддерживаемые ОС:** Windows, Linux, macOS.
*   **Сборка:** Native binaries (исполняемые файлы).

---

## 3. Функциональные требования (Functional Requirements)

### 3.1. Режимы работы
Утилита должна поддерживать два сценария использования:
1.  **Интерактивный режим:**
    *   Запускается, если пользователь не передал аргументов.
    *   Запрашивает название города: `Enter city name:`.
    *   Осуществляет поиск координат города через Geocoding API (Open-Meteo).
    *   Выводит результат.
2.  **Режим аргументов командной строки:**
    *   Запускается командой: `sky <city_name>`.
    *   Пропускает этап интерактивного ввода.
    *   Выводит результат.

### 3.2. Вывод данных (Output)
*   **Формат:** Чистый текст с поддержкой Эмодзи.
*   **Язык интерфейса:** Английский.
*   **Единицы измерения:** Метрическая система (Температура в °C, Скорость ветра в м/с).
*   **Поля данных:**
    *   Город (найденное название).
    *   Описание состояния (Clear, Cloudy, Rain и т.д.) + соответствующий эмодзи.
    *   Текущая температура.
    *   Ощущается как (Apparent temperature).

### 3.3. Обработка ошибок
*   Если город не найден или ошибка сети — выводить сообщение в `stderr` (например, `Error: City not found`).
*   Завершать работу с кодом возврата `1` (Error) при неудаче.
*   Завершать работу с кодом возврата `0` (Success) при успешном выводе.

### 3.4. Поведение
*   Утилита **не должна** создавать конфигурационные файлы на диске.
*   Утилита **не должна** сохранять историю запросов (stateless).
*   Кэширование не требуется (запрос идет каждый раз заново).

---

## 4. Пользовательские сценарии (User Stories)

### Сценарий А: Быстрая проверка погоды (Пользователь)
1.  Пользователь открывает терминал.
2.  Вводит команду: `sky London`.
3.  Видит результат:
    ```text
    London, GB
    Clear ☀️
    Temp: 15°C
    Feels like: 14°C
    ```

### Сценарий Б: Интерактивный запуск (Новичок)
1.  Пользователь просто вводит `sky`.
2.  Терминал показывает: `Enter city name:`.
3.  Пользователь вводит: `Paris`.
4.  Получает погоду для Парижа.

### Сценарий В: Ошибка (Скрипт/Автоматизация)
1.  Пользователь запускает `sky NonExistCity`.
2.  В `stderr` выводится: `Error: Location not found`.
3.  Команда завершается с кодом 1.

---

## 5. Требования к распространению (Distribution)
*   **Репозиторий:** GitHub.
*   **Релизы:** Автоматическая сборка бинарников под `windows-amd64`, `darwin-amd64`, `darwin-arm64` (Apple Silicon), `linux-amd64`.
*   **Установка:** Пользователь скачивает файл, переименовывает его (при необходимости) в `sky` (или `sky.exe`) и добавляет в переменную окружения `PATH`.

---

## 6. Дизайн (Макет вывода)

Пример того, как выглядит вывод в терминале:

```text
$ sky Tokyo

Feels like: 17°C
```
